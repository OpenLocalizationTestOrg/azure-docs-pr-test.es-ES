# <a name="manage-role-based-access-control"></a>Administrar el control de acceso basado en roles
En Azure Stack, un usuario puede ser un lector, propietario o colaborador de cada una de las instancias de una suscripción, grupo de recursos o servicio. Por ejemplo, el usuario A podría tener permisos de lector tooSubscription 1, pero tener permisos de propietario tooVirtual 7 de la máquina.

* Lector: el usuario puede ver todo el contenido, pero no puede realizar cambios.
* Colaborador: Usuario puede administrarlo todo excepto tooresources de acceso.
* Propietario: Usuario puede administrarlo todo, incluidos los tooresources de acceso.

## <a name="set-access-permissions-for-a-user"></a>Establecimiento de los permisos de acceso de un usuario
1. Inicie sesión con una cuenta que tenga recursos de toohello de permisos de propietario que desee toomanage.
2. En la hoja de hello para el recurso de hello, haga clic en hello **acceso** icono ![](media/azure-stack-manage-permissions/image1.png).
3. Hola **usuarios** hoja, haga clic en **Roles**.
4. Hola **Roles** hoja, haga clic en **agregar** tooadd permisos de usuario de Hola.
