## <a name="cluster-support"></a>Compatibilidad con clústeres
Servidor de copia de seguridad de Azure puede proteger los datos en hello después de aplicaciones en clúster:

-   Servidores de archivos

-   SQL Server

-   Hyper-V - si proteger un clúster de Hyper-V con protección de DPM de escala horizontal, no se puede agregar protección secundaria para las cargas de trabajo de Hyper-V de hello protegido.

    Si ejecuta Hyper-V en Windows Server 2008 R2, asegúrese de que la actualización de hello tooinstall descrito en KB [975354](https://support.microsoft.com/en-us/kb/975354).
    Si se ejecuta Hyper-V en Windows Server 2008 R2 en una configuración de clúster, asegúrese de instalar SP2 y el archivo del artículo KB [971394](https://support.microsoft.com/en-us/kb/971394).

-   Exchange Server: Azure Backup Server puede proteger clústeres de discos no compartidos para versiones de Exchange Server admitidas (replicación continua en clúster) y también puede proteger instancias de Exchange Server configuradas para replicación continua local.

-   SQL Server : Azure Backup Server no admite la realización de copias de seguridad de bases de datos de SQL Server hospedadas en volúmenes compartidos de clúster (CSV).

Servidor de copia de seguridad de Azure puede proteger las cargas de trabajo de clúster que se encuentran en hello mismo dominio como servidor DPM de hello o en un control secundario o dominio de confianza. Si desea tooprotect orígenes de datos en grupos de trabajo o dominios de confianza, utilice la autenticación NTLM o certificado para un único servidor o la autenticación de certificado solo para un clúster.
