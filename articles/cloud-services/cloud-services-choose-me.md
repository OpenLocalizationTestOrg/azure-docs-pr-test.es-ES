<span data-ttu-id="fb16a-144">Servicios en la nube también brinda supervisión.</span><span class="sxs-lookup"><span data-stu-id="fb16a-144">Cloud Services also provides monitoring.</span></span> <span data-ttu-id="fb16a-145">Al igual que con Azure Virtual Machines, detecta un servidor físico con errores y reinicia las VM que se ejecutaban en ese servidor en una máquina nueva.</span><span class="sxs-lookup"><span data-stu-id="fb16a-145">Like Azure Virtual Machines, it detects a failed physical server and restarts the VMs that were running on that server on a new machine.</span></span> <span data-ttu-id="fb16a-146">Pero Servicios en la nube también detecta máquinas virtuales y aplicaciones con errores, no solo errores de hardware.</span><span class="sxs-lookup"><span data-stu-id="fb16a-146">But Cloud Services also detects failed VMs and applications, not just hardware failures.</span></span> <span data-ttu-id="fb16a-147">A diferencia de Máquinas virtuales, Servicios en la nube tiene un agente dentro de cada rol web o rol de trabajo, por lo que puede iniciar nuevas instancias de aplicaciones y máquinas virtuales cuando se produce un error.</span><span class="sxs-lookup"><span data-stu-id="fb16a-147">Unlike Virtual Machines, it has an agent inside each web and worker role, and so it's able to start new VMs and application instances when failures occur.</span></span>

no hay nada parecido a un disco de datos de Máquinas virtuales.</span><span class="sxs-lookup"><span data-stu-id="fb16a-151">Unlike VMs created with Azure Virtual Machines, writes made to Cloud Services VMs aren't persistent;
