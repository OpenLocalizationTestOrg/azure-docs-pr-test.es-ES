<span data-ttu-id="b6363-122">Las importaciones y exportaciones solo se producen cuando se programan, lo que permite un mejor aislamiento de los cambios que ocurren en el sistema, dado que estos no se propagan automáticamente al origen de datos conectado.</span><span class="sxs-lookup"><span data-stu-id="b6363-122">Imports and exports only occur when scheduled, allowing for further insulation from changes occurring within the system, since changes do not automatically propagate to the connected data source.</span></span> <span data-ttu-id="b6363-123">Además, los desarrolladores pueden crear también sus propios conectores para conectarse a prácticamente cualquier origen de datos.</span><span class="sxs-lookup"><span data-stu-id="b6363-123">In addition, developers may also create their own connectors for connecting to virtually any data source.</span></span>

## <a name="connector-space"></a><span data-ttu-id="b6363-134">Espacio del conector</span><span class="sxs-lookup"><span data-stu-id="b6363-134">Connector space</span></span>

<span data-ttu-id="b6363-136">Esto permite que el servicio de sincronización funcione localmente sin necesidad de ponerse en contacto con el sistema remoto al sincronizar los objetos y restringe la interacción solo a las importaciones y exportaciones.</span><span class="sxs-lookup"><span data-stu-id="b6363-136">This allows the sync service to operate locally without the need to contact the remote system when synchronizing the objects and restricts interaction to imports and exports only.</span></span>

<span data-ttu-id="b6363-137">Cuando el origen de datos y el conector tienen la capacidad de proporcionar una lista de cambios (una importación delta), la eficiencia operativa aumenta significativamente porque solo se intercambian los cambios desde el último ciclo de sondeo.</span><span class="sxs-lookup"><span data-stu-id="b6363-137">When the data source and the connector have the ability to provide a list of changes (a delta import), then the operational efficiency increases dramatically as only changes since the last polling cycle are exchanged.</span></span> <span data-ttu-id="b6363-138">El espacio del conector aísla el origen de datos conectado de los cambios que se propagan automáticamente solicitando la importación y exportación de la programación del conector.</span><span class="sxs-lookup"><span data-stu-id="b6363-138">The connector space insulates the connected data source from changes propagating automatically by requiring that the connector schedule imports and exports.</span></span> <span data-ttu-id="b6363-139">Esta seguridad adicional le proporciona tranquilidad durante las pruebas, la obtención de vista previa o la confirmación de la próxima actualización.</span><span class="sxs-lookup"><span data-stu-id="b6363-139">This added insurance grants you peace of mind while testing, previewing, or confirming the next update.</span></span>

## <a name="metaverse"></a><span data-ttu-id="b6363-140">Metaverso</span><span class="sxs-lookup"><span data-stu-id="b6363-140">Metaverse</span></span>
<span data-ttu-id="b6363-141">El metaverso es la vista consolidada de todas las identidades combinadas de otros espacios de conector.</span><span class="sxs-lookup"><span data-stu-id="b6363-141">The metaverse is the consolidated view of all joined identities from neighboring connector spaces.</span></span>

<span data-ttu-id="b6363-142">Puesto que las identidades están vinculadas entra sí y la autoridad está asignada para varios atributos a través de asignaciones de flujo de importación, el objeto de metaverso central comienza a agregar información desde varios sistemas.</span><span class="sxs-lookup"><span data-stu-id="b6363-142">As identities are linked together and authority is assigned for various attributes through import flow mappings, the central metaverse object begins to aggregate information from multiple systems.</span></span> <span data-ttu-id="b6363-143">A partir de este flujo de atributos de objetos, las asignaciones proporcionan información a los sistemas de salida.</span><span class="sxs-lookup"><span data-stu-id="b6363-143">From this object attribute flow, mappings carry information to outbound systems.</span></span>

<span data-ttu-id="b6363-144">Los objetos se crean cuando un sistema autorizado los proyecta en el metaverso.</span><span class="sxs-lookup"><span data-stu-id="b6363-144">Objects are created when an authoritative system projects them into the metaverse.</span></span> <span data-ttu-id="b6363-145">En cuanto se quitan todas las conexiones, se elimina el objeto de metaverso.</span><span class="sxs-lookup"><span data-stu-id="b6363-145">As soon as all connections are removed, the metaverse object is deleted.</span></span>

<span data-ttu-id="b6363-146">Los objetos del metaverso no se pueden editar directamente.</span><span class="sxs-lookup"><span data-stu-id="b6363-146">Objects in the metaverse cannot be edited directly.</span></span> <span data-ttu-id="b6363-147">Todos los datos del objeto se deben aportar mediante el flujo de atributos.</span><span class="sxs-lookup"><span data-stu-id="b6363-147">All data in the object must be contributed through attribute flow.</span></span> <span data-ttu-id="b6363-148">El metaverso mantiene conectores persistentes con cada espacio de conector.</span><span class="sxs-lookup"><span data-stu-id="b6363-148">The metaverse maintains persistent connectors with each connector space.</span></span> <span data-ttu-id="b6363-149">Estos conectores no requieren una nueva evaluación en cada ejecución de la sincronización.</span><span class="sxs-lookup"><span data-stu-id="b6363-149">These connectors do not require reevaluation for each synchronization run.</span></span> <span data-ttu-id="b6363-150">Esto significa que Azure AD Connect Sync no tiene que buscar sistemáticamente el objeto remoto coincidente.</span><span class="sxs-lookup"><span data-stu-id="b6363-150">This means that Azure AD Connect sync does not have to locate the matching remote object each time.</span></span> <span data-ttu-id="b6363-151">De esta forma se evita la necesidad de costosos agentes para impedir cambios en los atributos que normalmente serían responsable de la correlación de los objetos.</span><span class="sxs-lookup"><span data-stu-id="b6363-151">This avoids the need for costly agents to prevent changes to attributes that would normally be responsible for correlating the objects.</span></span>

<span data-ttu-id="b6363-152">Durante la detección de nuevos orígenes de datos que pueden tener objetos ya existentes que deban administrarse, la sincronización de Azure AD Connect usa un proceso llamado regla de combinación para evaluar los posibles candidatos con los que se va a establecer un vínculo.</span><span class="sxs-lookup"><span data-stu-id="b6363-152">When discovering new data sources that may have preexisting objects that need to be managed, Azure AD Connect sync uses a process called a join rule to evaluate potential candidates with which to establish a link.</span></span>
<span data-ttu-id="b6363-153">Una vez establecido el vínculo, esta evaluación no se repite y el flujo de atributos normal puede producirse entre el origen de datos conectado remoto y el metaverso.</span><span class="sxs-lookup"><span data-stu-id="b6363-153">Once the link is established, this evaluation does not reoccur and normal attribute flow can occur between the remote connected data source and the metaverse.</span></span>

## <a name="provisioning"></a><span data-ttu-id="b6363-154">Aprovisionamiento</span><span class="sxs-lookup"><span data-stu-id="b6363-154">Provisioning</span></span>
<span data-ttu-id="b6363-155">Cuando un origen autoritativo proyecta un nuevo objeto en el metaverso, puede crearse un nuevo objeto de espacio de conector en otro conector que representa un origen de datos conectado de nivel inferior.</span><span class="sxs-lookup"><span data-stu-id="b6363-155">When an authoritative source projects a new object into the metaverse a new connector space object can be created in another Connector representing a downstream connected data source.</span></span>
