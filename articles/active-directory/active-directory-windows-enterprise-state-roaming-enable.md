## <a name="data-retention"></a><span data-ttu-id="d599a-132">Retención de datos</span><span class="sxs-lookup"><span data-stu-id="d599a-132">Data retention</span></span>
<span data-ttu-id="d599a-133">Los datos sincronizados con Azure a través de Enterprise State Roaming se conservarán indefinidamente a menos que se realice una operación de eliminación manual o se determine que los datos en cuestión sean obsoletos.</span><span class="sxs-lookup"><span data-stu-id="d599a-133">Data synced to Azure via Enterprise State Roaming will be retained indefinitely unless a manual delete operation is performed or the data in question is determined to be stale.</span></span> 

<span data-ttu-id="d599a-134">**Eliminación explícita:** los datos se eliminan cuando un administrador de Azure elimina un usuario o un directorio, o bien cuando un administrador solicita expresamente que se eliminen los datos.</span><span class="sxs-lookup"><span data-stu-id="d599a-134">**Explicit deletion:** The data is deleted when an Azure admin deletes a user or a directory or an admin requests explicitly that data is to be deleted.</span></span>

* <span data-ttu-id="d599a-135">**Eliminación de usuarios**: cuando se suprime un usuario en Azure AD, los datos de itinerancia de la cuenta de usuario se marcarán para eliminarse y la operación se completa entre 90 y 180 días.</span><span class="sxs-lookup"><span data-stu-id="d599a-135">**User deletion**: When a user is deleted in Azure AD, the user account roaming data will be marked for deletion and will be deleted between 90 to 180 days.</span></span> 
* <span data-ttu-id="d599a-136">**Eliminación de inquilinos**: la eliminación de un directorio completo en Azure AD se realiza inmediatamente.</span><span class="sxs-lookup"><span data-stu-id="d599a-136">**Directory deletion**: Deleting an entire directory in Azure AD is an immediate operation.</span></span> <span data-ttu-id="d599a-137">Todos los datos de configuración asociados a ese directorio se marcarán para eliminarse y la operación se completa entre 90 y 180 días.</span><span class="sxs-lookup"><span data-stu-id="d599a-137">All the settings data associated with that directory will be marked for deletion and will be deleted between 90 to 180 days.</span></span> 
* <span data-ttu-id="d599a-138">**Eliminación bajo demanda**: si el administrador de Azure AD desea eliminar manualmente los datos de configuración o la información de un usuario específico, el administrador puede generar una incidencia con el servicio de [soporte técnico de Azure](https://azure.microsoft.com/support/).</span><span class="sxs-lookup"><span data-stu-id="d599a-138">**On request deletion**: If the Azure AD admin wants to manually delete a specific user’s data or settings data, the admin can file a ticket with [Azure support](https://azure.microsoft.com/support/).</span></span> 

<span data-ttu-id="d599a-139">**Eliminación de datos obsoletos**: los datos a los que no se ha accedido durante un año (el periodo de retención) se considerarán obsoletos y se podrán eliminar de Azure.</span><span class="sxs-lookup"><span data-stu-id="d599a-139">**Stale data deletion**: Data that has not been accessed for one year (“the retention period”) will be treated as stale and may be deleted from Azure.</span></span> <span data-ttu-id="d599a-140">El periodo de retención está sujeto a cambios, pero no será inferior a 90 días.</span><span class="sxs-lookup"><span data-stu-id="d599a-140">The retention period is subject to change but will not be less than 90 days.</span></span> <span data-ttu-id="d599a-141">Los datos obsoletos pueden ser un conjunto específico de valores de configuración de Windows o de la aplicación o toda la configuración de un usuario.</span><span class="sxs-lookup"><span data-stu-id="d599a-141">The stale data may be a specific set of Windows/application settings or all settings for a user.</span></span> <span data-ttu-id="d599a-142">Por ejemplo:</span><span class="sxs-lookup"><span data-stu-id="d599a-142">For example:</span></span>

* <span data-ttu-id="d599a-143">Si ningún dispositivo accede a una colección de configuraciones concreta (por ejemplo, una aplicación se quita del dispositivo o un grupo de configuraciones, como "Tema", está deshabilitado para todos los dispositivos de un usuario), esa colección quedará obsoleta tras el período de retención y se puede eliminar.</span><span class="sxs-lookup"><span data-stu-id="d599a-143">If no devices access a particular settings collection (e.g., an application is removed from the device, or a settings group such as “Theme” is disabled for all of a user’s devices), then that collection will become stale after the retention period and may be deleted.</span></span> 
* <span data-ttu-id="d599a-144">Si un usuario ha desactivado la sincronización de la configuración en todos sus dispositivos, no se podrá acceder a ninguno de los datos de configuración y todos los datos de configuración para ese usuario quedarán obsoletos y pueden eliminarse tras el período de retención.</span><span class="sxs-lookup"><span data-stu-id="d599a-144">If a user has turned off settings sync on all his/her devices, then none of the settings data will be accessed, and all the settings data for that user will become stale and may be deleted after the retention period.</span></span> 
* <span data-ttu-id="d599a-145">Si el administrador del directorio de Azure AD desactiva Enterprise State Roaming para todo el directorio, todos los usuarios de ese directorio dejarán de sincronizar la configuración y todos los datos de configuración de todos los usuarios quedarán obsoletos y podrán eliminarse tras el período de retención.</span><span class="sxs-lookup"><span data-stu-id="d599a-145">If the Azure AD directory admin turns off Enterprise State Roaming for the entire directory, then all users in that directory will stop syncing settings, and all settings data for all users will become stale and may be deleted after the retention period.</span></span> 

<span data-ttu-id="d599a-146">**Recuperación de los datos eliminados**: la directiva de retención de datos no es configurable.</span><span class="sxs-lookup"><span data-stu-id="d599a-146">**Deleted data recovery**: The data retention policy is not configurable.</span></span> <span data-ttu-id="d599a-147">Cuando los datos se hayan eliminado permanentemente, no se pueden recuperar.</span><span class="sxs-lookup"><span data-stu-id="d599a-147">Once the data has been permanently deleted, it will not be recoverable.</span></span> <span data-ttu-id="d599a-148">Sin embargo, es importante tener en cuenta que solo se eliminarán los datos de configuración de Azure, no el dispositivo del usuario final.</span><span class="sxs-lookup"><span data-stu-id="d599a-148">However, it’s important to note that the settings data will only be deleted from Azure, not the end-user device.</span></span> <span data-ttu-id="d599a-149">Si cualquier dispositivo más adelante vuelve a conectarse al servicio Enterprise State Roaming, la configuración se sincronizará de nuevo y almacenará en Azure.</span><span class="sxs-lookup"><span data-stu-id="d599a-149">If any device later reconnects to the Enterprise State Roaming service, the settings will again be synced and stored in Azure.</span></span>
