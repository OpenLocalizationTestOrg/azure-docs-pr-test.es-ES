## <a name="how-does-automated-provisioning-work"></a><span data-ttu-id="1680d-140">¿Cómo funciona el aprovisionamiento automático?</span><span class="sxs-lookup"><span data-stu-id="1680d-140">How Does Automated Provisioning Work?</span></span>
<span data-ttu-id="1680d-141">Azure AD aprovisiona a los usuarios para aplicaciones SaaS mediante la conexión de los extremos de aprovisionamiento proporcionados por el proveedor de cada aplicación.</span><span class="sxs-lookup"><span data-stu-id="1680d-141">Azure AD provisions users to SaaS apps by connecting to provisioning endpoints provided by each application vendor.</span></span> <span data-ttu-id="1680d-142">Estos extremos permiten a Azure AD crear, actualizar y quitar usuarios mediante programación.</span><span class="sxs-lookup"><span data-stu-id="1680d-142">These endpoints allow Azure AD to programmatically create, update, and remove users.</span></span> <span data-ttu-id="1680d-143">A continuación se ofrece una breve introducción de los diferentes pasos que Azure AD realiza para automatizar el aprovisionamiento.</span><span class="sxs-lookup"><span data-stu-id="1680d-143">Below is a brief overview of the different steps that Azure AD takes to automate provisioning.</span></span>

1. <span data-ttu-id="1680d-144">Al habilitar el aprovisionamiento de una aplicación por primera vez, se realizan las siguientes acciones:</span><span class="sxs-lookup"><span data-stu-id="1680d-144">When you enable provisioning for an application for the first time, the following actions are performed:</span></span>
   * <span data-ttu-id="1680d-145">Azure AD intentará hacer coincidir todos los usuarios existentes en la aplicación SaaS con sus identidades correspondientes en el directorio.</span><span class="sxs-lookup"><span data-stu-id="1680d-145">Azure AD will attempt to match any existing users in the SaaS app with their corresponding identities in the directory.</span></span> <span data-ttu-id="1680d-146">Cuando se asocia un usuario, *no* se habilitan automáticamente para el inicio de sesión único.</span><span class="sxs-lookup"><span data-stu-id="1680d-146">When a user is matched, they are *not* automatically enabled for single sign-on.</span></span> <span data-ttu-id="1680d-147">Para que un usuario disponga de acceso a la aplicación, deben asignarse explícitamente a la aplicación en Azure AD, ya sea directamente o a través de la pertenencia a grupos.</span><span class="sxs-lookup"><span data-stu-id="1680d-147">In order for a user to have access to the application, they must be explicitly assigned to the app in Azure AD, either directly or via group membership.</span></span>
   * <span data-ttu-id="1680d-148">Si ya ha especificado qué usuarios deben asignarse a la aplicación, y si Azure AD no logra encontrar cuentas existentes para tales usuarios, Azure AD aprovisionará cuentas nuevas para ellos en la aplicación.</span><span class="sxs-lookup"><span data-stu-id="1680d-148">If you have already specified which users should be assigned to the application, and if Azure AD fails to find existing accounts for those users, Azure AD will provision new accounts for them in the application.</span></span>
2. <span data-ttu-id="1680d-149">Una vez que se ha completado la sincronización inicial, como se ha descrito anteriormente, Azure AD comprobará cada 10 minutos si se han producido los siguientes cambios:</span><span class="sxs-lookup"><span data-stu-id="1680d-149">Once the initial synchronization has been completed as described above, Azure AD will check every 10 minutes for the following changes:</span></span>
   * <span data-ttu-id="1680d-150">Si se han asignado usuarios nuevos a la aplicación (directamente o a través de la pertenencia a grupos), se les aprovisionará una cuenta nueva en la aplicación SaaS.</span><span class="sxs-lookup"><span data-stu-id="1680d-150">If new users have been assigned to the application (either directly or through group membership), then they will be provisioned a new account in the SaaS app.</span></span>
   * <span data-ttu-id="1680d-151">Si se ha anulado el acceso de un usuario, su cuenta en la aplicación SaaS se marcará como deshabilitada (los usuarios nunca se eliminarán totalmente, lo que lo protege frente a pérdidas de datos si se producen errores de configuración).</span><span class="sxs-lookup"><span data-stu-id="1680d-151">If a user's access has been removed, then their account in the SaaS app will be marked as disabled (users are never fully deleted, which protects you from data loss in the event of a misconfiguration).</span></span>
   * <span data-ttu-id="1680d-152">Si ha asignado un usuario a la aplicación recientemente y ya tenía una cuenta en la aplicación SaaS, esa cuenta se marcará como habilitada, y algunas propiedades de usuario pueden actualizarse si están obsoletas con respecto al directorio.</span><span class="sxs-lookup"><span data-stu-id="1680d-152">If a user was recently assigned to the application and they already had an account in the SaaS app, that account will be marked as enabled, and certain user properties may be updated if they are out-of-date compared to the directory.</span></span>
   * <span data-ttu-id="1680d-153">Si se ha cambiado la información del usuario (por ejemplo, el número de teléfono, el domicilio de la oficina, etc.) en el directorio, esta información también se actualizará en la aplicación SaaS.</span><span class="sxs-lookup"><span data-stu-id="1680d-153">If a user's information (such as phone number, office location, etc) has been changed in the directory, then that information will also be updated in the SaaS application.</span></span>

<span data-ttu-id="1680d-154">Para obtener más información sobre cómo se asignan los atributos entre Azure AD y la aplicación SaaS, consulte el artículo [Personalización de asignaciones de atributos](active-directory-saas-customizing-attribute-mappings.md).</span><span class="sxs-lookup"><span data-stu-id="1680d-154">For more information on how attributes are mapped between Azure AD and your SaaS app, see the article on [Customizing Attribute Mappings](active-directory-saas-customizing-attribute-mappings.md).</span></span>
